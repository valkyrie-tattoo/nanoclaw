# Deployment Pipeline

Deploy projects to production with pre-deploy verification, monitoring, and GEO baseline testing.

## When to Use

- Phase 5 (Deploy) of any project
- Deploying updates to an existing site
- Setting up monitoring for a launched site
- Jesse says "deploy", "ship it", "push to production"

## Prerequisites

**First-time setup (per container session):**
```bash
which vercel || npm i -g vercel
```

**Environment:**
- `$VERCEL_TOKEN` must be set (injected by host)
- If missing, flag to Jesse: "Need VERCEL_TOKEN in .env to deploy"

## Preview Deploy

ALWAYS preview before production. Jesse approves the preview URL before go-live.

```bash
cd /workspace/group/projects/{slug}/src

# Build first
npm run build

# Deploy to preview
vercel deploy --token=$VERCEL_TOKEN --yes 2>&1
```

This returns a preview URL like `https://project-abc123.vercel.app`.

**Send preview URL to Jesse:**
Use `send_message` with the preview URL and a note about what to check.

**Wait for Jesse's approval before production deploy.**

## Production Deploy

Only after Jesse approves the preview:

```bash
cd /workspace/group/projects/{slug}/src

vercel deploy --prod --token=$VERCEL_TOKEN --yes 2>&1
```

**Post-deploy verification:**
```bash
# Verify site is live
agent-browser open {production-url}
agent-browser wait --load networkidle
agent-browser screenshot -o .lighthouse/post-deploy-verification.png

# Run quick Lighthouse check
npx lighthouse {production-url} --output=json --output-path=.lighthouse/post-deploy.json --chrome-flags="--headless --no-sandbox"
```

## Environment Variables

Set project-specific env vars via Vercel CLI:

```bash
# Set a variable
echo "value" | vercel env add VARIABLE_NAME production --token=$VERCEL_TOKEN

# List current vars
vercel env ls --token=$VERCEL_TOKEN
```

Common env vars per project:
- `NEXT_PUBLIC_SITE_URL` — canonical URL
- `PLAUSIBLE_DOMAIN` or `FATHOM_SITE_ID` — analytics
- Any API keys for forms, CRM, etc.

## Domain Configuration

Vercel handles DNS for custom domains:

```bash
# Add custom domain
vercel domains add example.com --token=$VERCEL_TOKEN

# Check DNS status
vercel domains inspect example.com --token=$VERCEL_TOKEN
```

**Guide Jesse through DNS setup** if needed — provide the exact DNS records (CNAME or A record) to set at their registrar.

## Rollback

If something breaks after production deploy:

```bash
# List recent deployments
vercel ls --token=$VERCEL_TOKEN

# Promote a previous deployment to production
vercel promote {deployment-url} --token=$VERCEL_TOKEN
```

Zero-downtime rollback. The previous deployment becomes production instantly.

## Search Console Submission

After production deploy:

1. **Generate sitemap** (should be auto-generated by framework):
   - Astro: `@astrojs/sitemap` integration
   - Next.js: `next-sitemap` package or app router metadata
   - Static: generate `sitemap.xml` manually

2. **Verify sitemap is accessible:**
```bash
agent-browser open {production-url}/sitemap.xml
agent-browser screenshot -o .lighthouse/sitemap-verification.png
```

3. **Submit to Search Console:** Guide Jesse to add the property in Google Search Console and submit the sitemap URL. Loom cannot directly access Search Console.

4. **Verify robots.txt:**
```bash
agent-browser open {production-url}/robots.txt
agent-browser eval "document.body.innerText"
```

Ensure it contains:
```
User-agent: *
Allow: /
Sitemap: {production-url}/sitemap.xml
```

## Monitoring Setup

**Uptime monitoring:**
- Recommend UptimeRobot (free tier: 50 monitors, 5-min checks) or Better Uptime
- Guide Jesse through setup — add the production URL as a monitor
- Set up alert notification to Jesse's email/phone

**Core Web Vitals:**
- After 28 days of traffic, CrUX data becomes available
- Check via PageSpeed Insights: `https://pagespeed.web.dev/analysis?url={production-url}`
- Flag any metrics in "needs improvement" or "poor" range

**SSL certificate:**
- Vercel handles SSL automatically (Let's Encrypt)
- Verify: `agent-browser eval "location.protocol"` should return `https:`

**Broken link detection:**
- Schedule periodic crawl (monthly) to check all links
- Flag any 404s for immediate fix

## GEO Baseline Test

After production deploy, run the GEO baseline test from `loom-geo` skill:
- Query AI engines (Perplexity, ChatGPT, Gemini) for relevant industry + location terms
- Record whether the site appears in responses
- Save baseline to `.lighthouse/geo-baseline.md`
- This is the starting point for tracking GEO improvement over time

## Alternative Deploy Targets

**Netlify:**
```bash
npm i -g netlify-cli
netlify deploy --dir=dist --prod --auth=$NETLIFY_TOKEN
```

**Cloudflare Pages:**
```bash
npm i -g wrangler
wrangler pages deploy dist --project-name={slug}
```

Use only if Jesse specifies. Default is always Vercel.
